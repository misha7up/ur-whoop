# UREMONT WHOOP — Покрытие блоков управления (ECU)

> Версия 1.7.0-stable (build 7) | 21 марочная группа (20 именованных + OTHER) | актуальность сверена с кодом 2026-04-03
> Покрывает **все топ-15 марок автопарка РФ** + ведущие китайские бренды + Subaru

---

## Универсальные блоки (все марки)

Опрашиваются **всегда**, через стандартные OBD2 CAN-адреса:

| Блок | CAN-адрес (ATSH) | Режимы |
|------|-------------------|--------|
| **Двигатель / ЭБУ** | `7E0` (по умолчанию) | Mode 01–09, 0A |
| **ABS / Тормоза** | `7B0` | Mode 03 / 07 / 0A |
| **SRS / Подушки безопасности** | `7D0` | Mode 03 / 07 / 0A |
| **Коробка передач (TCM)** | `7E1` | Mode 03 / 07 / 0A |
| **Доп. силовой (гибрид / дизель / 2-й ECM)** | `7E2` | Mode 03 / 07 / 0A |
| **Раздаточная коробка / 4WD** | `7E3` | Mode 03 / 07 / 0A |
| **Кузов (BCM)** | `7E4` | Mode 03 / 07 / 0A |

`7E2` — гибридная батарея (Toyota/Hyundai HEV/PHEV), дизельное дожигание (DPF/SCR), вторичный ECM.
`7E3` — раздаточная коробка (4WD/AWD), вторичный модуль (GM Transfer Case и др.).

Список опроса `ecuProbeList` склеивает строку WMI (`detectedMake` из ЭБУ) и **поле «марка» из ручного профиля** (`mergeMakeHints`). Для каждой строки из `ALL_MAKES` (Android `MainActivity`, iOS `DtcDatabase`) марочные адреса подмешиваются так:

## Примечания

- **Mode 03 / 07 / 0A** на доп. блоках: **07 и 0A** выполняются только если блок ответил на Mode 03 без ошибки (тот же шаг, что и для моторного сценария). Если подтверждённые коды пришли **только через UDS 0x19** (после сбоя Mode 03), pending/permanent для этого адреса **не** опрашиваются — в отчёте только список из UDS.
- **UDS 0x19 fallback** — если Mode 03 вернул `NO DATA` или `7F 03 11` (Service Not Supported), автоматически пробуется UDS `10 03` (Extended Session) + `19 02 FF` (ReadDTCByStatusMask). DTC кодируются 3 байтами (J2012) + 1 байт Failure Type Byte (FTB); формат: `P0420-17` (с суффиксом FTB) или `P0420` (FTB=00). Только CAN.
- **Одометр UDS (0x22)** — экспериментальный, работает только при CAN-протоколе (ISO 15765)
- **«prelude 1003»** — перед запросом одометра / UDS 0x19 отправляется DiagnosticSessionControl Extended Session
- Блоки без ECU на шине (`NO DATA` и на Mode 03, и на UDS) отображаются как «Блок не отвечает»
- Дубликаты `txHeader` между универсальными и марочными автоматически исключаются
- **K-Line fix**: На ISO 9141-2 / KWP2000 парсинг DTC не пропускает байт-счётчик (его нет на non-CAN)
- **UDS не пробуется на K-Line** — ATSH + UDS работает только при CAN-протоколе
